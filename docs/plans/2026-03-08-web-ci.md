# Web CI Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Add a GitHub Actions workflow that type-checks and builds the SvelteKit frontend
on every PR and push to main that touches `web/**`.

**Architecture:** Single workflow file mirroring the existing `firmware.yml` convention —
path-filtered triggers, SHA-pinned actions, npm cache via `actions/setup-node`, working
directory scoped to `web/`. No linting step (oxlint not yet installed).

**Tech Stack:** GitHub Actions, Node 22 LTS, npm, SvelteKit (`svelte-check`, `vite build`)

---

### Task 1: Create the web CI workflow

**Files:**
- Create: `.github/workflows/web.yml`

**Step 1: Create the workflow file**

```yaml
name: Web

on:
  pull_request:
    paths:
      - 'web/**'
      - '.github/workflows/web.yml'
  push:
    branches: [main]
    paths:
      - 'web/**'
      - '.github/workflows/web.yml'

jobs:
  check:
    runs-on: ubuntu-24.04
    permissions:
      contents: read
    defaults:
      run:
        working-directory: web

    steps:
      - uses: actions/checkout@34e114876b0b11c390a56381ad16ebd13914f8d5  # v4

      - uses: actions/setup-node@49933ea5288caeca8642d1e84afbd3f7d6820020  # v4
        with:
          node-version: '22'
          cache: 'npm'
          cache-dependency-path: web/package-lock.json

      - name: Install dependencies
        run: npm ci

      - name: Type-check
        run: npm run check

      - name: Build
        run: npm run build
```

**Step 2: Validate YAML syntax**

```bash
actionlint .github/workflows/web.yml
```

Expected: no output (clean).

If `actionlint` isn't installed: `brew install actionlint && actionlint .github/workflows/web.yml`

**Step 3: Commit**

```bash
git add .github/workflows/web.yml
git commit -m "ci: add SvelteKit type-check and build workflow"
```
